Conceptos Básicos de TypeScript

¿Qué es TypeScript?
TypeScript es un lenguaje de programación de código abierto desarrollado por Microsoft. Es un superconjunto tipado de JavaScript, lo que significa que añade tipos estáticos opcionales a JavaScript, ayudando a detectar errores antes de ejecutar el código.

¿Qué ventajas ofrece TypeScript frente a JavaScript puro?
Ofrece detección temprana de errores durante el desarrollo, mejor autocompletado en los editores de código, facilita el mantenimiento y escalabilidad de aplicaciones grandes, y permite usar características modernas de JavaScript con soporte para navegadores más antiguos.

¿Qué extensión de archivo se utiliza para los archivos TypeScript?
La extensión estándar para los archivos de TypeScript es .ts. Si el archivo contiene sintaxis de React (JSX), la extensión utilizada es .tsx.

Estos son los conceptos fundamentales para entender qué es TypeScript y por qué se utiliza.
Instalación y Compilación

¿Cómo se instala TypeScript en VSCode?
TypeScript se instala globalmente en el sistema mediante Node.js y npm usando el comando npm install -g typescript en la terminal. VS Code ya trae soporte incorporado para TypeScript, por lo que una vez instalado el compilador, el editor lo reconocerá automáticamente.

¿Cómo se compila un archivo TypeScript para convertirlo a JavaScript?
Se compila usando el comando tsc seguido del nombre del archivo en la terminal. Por ejemplo: tsc archivo.ts. Esto generará un nuevo archivo archivo.js con el código equivalente en JavaScript puro que los navegadores pueden entender.

La configuración adecuada del entorno es esencial para trabajar con TypeScript.
Tipos de Datos en TypeScript

¿Cómo se define un tipo de dato en TypeScript?
Se define utilizando dos puntos (:) seguido del tipo de dato después del nombre de la variable. Por ejemplo: let edad: number = 25; o let nombre: string = "Juan";.

¿Qué son los tipos de unión y cómo se declaran?
Los tipos de unión permiten que una variable pueda almacenar más de un tipo de dato especificado. Se declaran utilizando el símbolo de tubería (|). Por ejemplo: let identificador: string | number;.

¿Qué es un tipo literal en TypeScript y para qué se utiliza?
Un tipo literal restringe una variable a un valor exacto o a un conjunto de valores específicos en lugar de un tipo general. Se utiliza frecuentemente junto con tipos de unión. Por ejemplo: let estado: "activo" | "inactivo";.

¿Qué son los tipos enumerados (Enums) y cómo se crean?
Los Enums permiten definir un conjunto de constantes con nombre, haciendo el código más legible e intencional. Se crean con la palabra clave enum. Por ejemplo: enum Direccion { Arriba, Abajo, Izquierda, Derecha }.

¿Cómo se define un tipo alias y cuándo es útil?
Un alias de tipo permite darle un nombre personalizado a un tipo existente usando la palabra clave type. Es muy útil para reutilizar tipos complejos (como objetos o uniones) en diferentes partes del código sin tener que reescribirlos. Ejemplo: type ID = string | number;.

El sistema de tipos es la característica principal de TypeScript.
Funciones en TypeScript

¿Cómo se definen las funciones con tipos en TypeScript?
Se pueden definir tipos tanto para los parámetros de entrada como para el valor de retorno de la función. Por ejemplo: function sumar(a: number, b: number): number { return a + b; }.

¿Qué son los parámetros opcionales y los parámetros predeterminados en funciones?
Los parámetros opcionales se indican con un signo de interrogación (?) al final del nombre y pueden omitirse al llamar a la función. Los parámetros predeterminados tienen un valor asignado directamente en la declaración (param: tipo = valor), que se usará si no se proporciona ese argumento.

Tipar correctamente las funciones previene muchos errores en tiempo de ejecución.
Mejores Prácticas

¿Cuáles son algunas mejores prácticas para escribir código limpio y mantenible en TypeScript?

  • Usar tipos estrictos y evitar usar el tipo any en lo posible.
  • Activar el modo estricto ("strict": true) en el archivo de configuración tsconfig.json.
  • Usar interfaces para definir la forma de los objetos.
  • Aprovechar los Enums o tipos de unión para valores restringidos.
  • Mantener funciones y clases pequeñas y con una única responsabilidad.
  • Usar nombres descriptivos para variables, funciones y tipos personalizados.

Aplicar mejores prácticas asegura que el código sea escalable y fácil de entender.
